# Basiselemente des Programmablaufs

Theoretisch könnten alle in der Programmierung möglichen Aufgabenstellungen
durch **elementare Anweisungen**, **Sequenzen** und **Schleifen** 
gelöst werden, denn durch diese Grundelemente ist es möglich die 
Reihenfolge, in der Anweisungen abgearbeitet werden, eindeutig zu kontrollieren.
Anweisungen, Sequenzen und Schleifen werden deshalb auch als Kontrollelemente bezeichnet. 
Alle weiteren Grundelemente erleichtern jedoch die Arbeit des Programmierens sehr.

### 1.) Elementare Anweisung

***Schneide das Fleisch in kleine Würfel***

Die elementare Anweisung ist das einfachste algorithmische Grundelement. 
Durch eine elementare Anweisung wird ein Computer veranlasst, 
eine bestimmte vordefinierte Aktion durchzuführen. 
Ein Programm kann als eine Abfolge von solchen Anweisungen gesehen werden. 
Anweisungen werden elementar genannt, wenn sie nicht weiter zerlegt werden können.

**Beispiele:**

- **Wertzuweisung** (x = 1) 
- **Addition** (10 + 20)
- **Vergleich** (name === "Hans")

### 2.) Anweisungsblock

***Bringe das Wasser zum Kochen, dann gib das Paket Nudeln hinein, 
schneide das Fleisch, dann das Gemüse ...***

Werden mehrere Anweisungen hintereinander ausgeführt, 
spricht man von einem **Anweisungsblock** oder auch einer **Sequenz** oder **Folge** von Anweisungen.

### 3.) Schleifen

***Rühre die Soße so lange, bis diese fest ist.***

**Beispiele:** 

- **for**
- **foreach** 
- **while**
- **do-while**

### 4.) Bedingte Ausführung (Zweigung)

***Wenn die Soße zu dünn ist, dann füge Mehl hinzu.***

**Beispiele:**

- **if**
- **else if**
- **switch**


### 5.) Parallele Ausführung

***Ich schneide das Fleisch, du das Gemüse, er stellt inzwischen Wasser auf, sie holt die Gewürze ...***

### 6.) Unterprogramm

***Bereite die Soße so, wie im Anhang erklärt wird.***

Die Tätigkeit wird anderswo beschrieben und ist mehrfach benutzbar.
